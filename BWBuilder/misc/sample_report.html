Issue#1 Logical Attacks: Abuse of Functionality
Vulnerable URL: http://crackme.cenzic.com/
 
Severity: Low
Reasoning: It was found possible to TRACE/TRACK the server.
Threat details:
Remediation:

Configure your web server properly to disallow unnecessary HTTP method


For apache web server, add the following lines for each virtual host in your configuration file :

RewriteEngine on

RewriteCond %{REQUEST_METHOD} ^(TRACE|TRACK)

RewriteRule .* - [F]


Alternatively, note that Apache versions 1.3.34, 2.0.55, and 2.2 support disabling the TRACE method natively via the 'TraceEnable' directive.Add TraceEnable off in the main server config file httpd.conf.


For Microsoft's IIS, install URLSCAN tool and do the following things Open urlscan.ini file(c:/Windows/Systems/intsrv/urlscan). Set Allowverbs=0 and Put Trace Deny verb Methods list and restart IIS .
References: http://www.kb.cert.org/vuls/id/867593
Variant 1 of 1
 
Description:

TRACE/TRACK method was enabled on the following URI:

http://crackme.cenzic.com/


An attack known as Cross Site Tracing (XST) leverages this method to steal cookies. Once a session key in a cookie is obtained by an attacker, he can hijack a legitimate user's session.
